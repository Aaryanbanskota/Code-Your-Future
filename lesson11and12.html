OOP Superpowers

Lesson 11 & 12: Inheritance, Polymorphism, Encapsulation, and Abstraction

Learn the four pillars of OOP with detailed examples and become a Java superhero!

🌟 Welcome back, Java Adventurer!

You've learned about Classes and Objects. That was like learning how to make toys from a mold. Now, it's time to learn how to reuse, extend, and protect those toys so you don't have to start from scratch every time.

This lesson is about 4 Superpowers of OOP (Object-Oriented Programming). They are:

Inheritance

Passing traits from parents to children 👨‍👩‍👧‍👦

Polymorphism

One name, many forms 🎭

Encapsulation

Keeping data safe 🔒

Abstraction

Showing only the important details 🎨

🧬 1. Inheritance — Passing Traits from Parents to Children

Inheritance means a class (child) can take all the features of another class (parent).

💡 Example in real life: A parent class "Animal" can have common traits like eat() and sleep(). A child class "Dog" or "Cat" can inherit these traits, but also add their own like bark() or meow().

Example: Inheritance in Java

// Parent class
class Animal {
    void eat() {
        System.out.println("This animal eats food 🍎");
    }
}

// Child class
class Dog extends Animal {
    void bark() {
        System.out.println("The dog barks 🐶");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog d = new Dog();
        d.eat();   // Inherited from Animal
        d.bark();  // Defined in Dog
    }
}

✅ Output:

This animal eats food 🍎
The dog barks 🐶

⚡ Key Idea: Inheritance lets you reuse code instead of writing the same thing again.

🎭 2. Polymorphism — One Name, Many Forms

Polymorphism means "many forms."

The same method can behave differently depending on which object is using it.

💡 Think of the word "play" — a child plays with toys, a singer plays music, a footballer plays football. Same word, different meaning!

Example: Method Overriding (Run-Time Polymorphism)

class Animal {
    void sound() {
        System.out.println("Animals make sounds 🎵");
    }
}

class Dog extends Animal {
    void sound() {
        System.out.println("Dog says Woof! 🐶");
    }
}

class Cat extends Animal {
    void sound() {
        System.out.println("Cat says Meow! 🐱");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal a1 = new Dog();  // Dog object
        Animal a2 = new Cat();  // Cat object
        
        a1.sound();  // Woof!
        a2.sound();  // Meow!
    }
}

✅ Output:

Dog says Woof! 🐶
Cat says Meow! 🐱

⚡ Key Idea: Polymorphism = One method name, different behaviors.

🔒 3. Encapsulation — Keep Data Safe

Encapsulation means hiding the details and protecting data.

You wrap variables inside a class and only allow access through methods (getters & setters).

💡 Just like medicine is kept in a capsule — you don't see what's inside, you just take it safely. 💊

Example: Encapsulation

class BankAccount {
    private int balance = 1000; // private: hidden from outside

    // Getter
    public int getBalance() {
        return balance;
    }

    // Setter
    public void deposit(int amount) {
        balance += amount;
    }
}

public class Main {
    public static void main(String[] args) {
        BankAccount account = new BankAccount();
        
        // Access balance safely
        System.out.println("Balance: " + account.getBalance());
        
        account.deposit(500);
        System.out.println("After deposit: " + account.getBalance());
    }
}

✅ Output:

Balance: 1000
After deposit: 1500

⚡ Key Idea: Encapsulation = Protecting variables by using private + getters/setters.

🎨 4. Abstraction — Show Only What's Important

Abstraction means hiding the complex stuff and showing only what is necessary.

💡 When you drive a car, you just use the steering wheel and pedals 🚗. You don't care about the engine's inner details.

Example: Abstract Class

abstract class Animal {
    abstract void sound(); // No body, only idea
}

class Dog extends Animal {
    void sound() {
        System.out.println("Dog says Woof! 🐶");
    }
}

class Cat extends Animal {
    void sound() {
        System.out.println("Cat says Meow! 🐱");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal d = new Dog();
        Animal c = new Cat();
        
        d.sound();
        c.sound();
    }
}

✅ Output:

Dog says Woof! 🐶
Cat says Meow! 🐱

⚡ Key Idea: Abstraction = Show only the important parts, hide the messy details.
