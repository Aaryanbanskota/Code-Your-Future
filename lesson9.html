Lesson 9: Introduction to Object-Oriented Programming (OOP)

Learn the fundamentals of Object-Oriented Programming in Java - classes, objects, and the four pillars of OOP!

So far, you've been writing programs that use variables, loops, methods, and arrays. But in the real world, programs get BIG, and handling everything with just those tools becomes messy.

üè† 1. What is OOP?

OOP = Object-Oriented Programming.

Think of the world around you üåç:

  • A Car has properties (color, brand, speed) and actions (drive, brake, honk).
  • A Dog has properties (name, breed, age) and actions (bark, eat, sleep).

In Java, we can represent these as objects inside our program.

üí° Instead of thinking "just code", OOP makes you think in terms of objects.

üß± 2. Four Pillars of OOP

OOP is built on four magic pillars:

Encapsulation

Wrapping variables and methods inside a class (like a capsule üíä).

Abstraction

Showing only the necessary details, hiding the rest (like a car steering wheel ‚Äî you don't see the engine's complexity).

Inheritance

One class can inherit properties and behaviors from another (like a child inheriting traits from parents üë®‚Äçüë©‚Äçüëß).

Polymorphism

One thing behaving in many forms (like the word run: run in sports, run a company, run a program üèÉ‚Äç‚ôÇÔ∏è).

üë®‚Äçüíª 3. Classes and Objects (The Basics of OOP)

A class is like a blueprint.

An object is a real thing created from that blueprint.

Example:

// Class (Blueprint)
class Car {
    String brand;
    int speed;

    void drive() {
        System.out.println(brand + " is driving at " + speed + " km/h");
    }
}

// Main Program
public class Main {
    public static void main(String[] args) {
        // Create an Object
        Car myCar = new Car();
        myCar.brand = "Tesla";
        myCar.speed = 120;

        // Call method
        myCar.drive();
    }
}

‚úÖ Output:

Tesla is driving at 120 km/h

üìù 4. Why OOP is Awesome?

  • Helps organize big projects.
  • Makes code reusable (write once, use many times).
  • Makes teamwork easier (different programmers work on different classes).
  • Reflects the real world, making code easier to understand.

üåà Summary

  • OOP = a way to organize programs using objects.
  • Class = blueprint, Object = actual item created from class.
  • OOP has 4 pillars: Encapsulation, Abstraction, Inheritance, Polymorphism.
  • OOP makes Java powerful, clean, and closer to real-world thinking.
